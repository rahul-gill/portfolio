Relational Algebra

  • relational algebra defines a set of operations for the relational model(gives operational description of a computation, on entire tables(sets of tuples), not on indivisual tuples or cells in them)
  • the relational calculus provides a higher-level declarative language for specifying relational queries(just like sql). One example {S ∣ S∈Sailors ∃R∈Reserves (R.sid=S.sid∧R.bid=103)}\{S\ | \ S\in Sailors\ \exist R\in Reserves\ (R.sid= S.sid \land R.bid = 103) \}{S ∣ S∈Sailors ∃R∈Reserves (R.sid=S.sid∧R.bid=103)}
  • Codd's theorem established equivalence in expressivity between these two


  • algebra of opeartors on relational instances

    • closed: result of operations is also an relational instances
    • typed: input schema determines output schema(so query's legality can be checked statically)
    • no duplicate tuples in pure relational algebra

  • Unary Operators

    • Projection(π\piπ): retain only desired columns(vertical)
    • Selection(σ\sigmaσ): retain subset of rows(horizontal)
    • Renaming(ρ\rhoρ): rename attributes and relations

  • Binary Operators

    • Union(∪\cup∪)
    • Set difference(−-−)
    • Cartesian Product(×\times×)

  • Compound operators: macros for the above

    • Intersection(∩\cap∩)
    • Joins(⋈,⋈θ\bowtie, \bowtie_\theta⋈,⋈θ​): combine relations that satisfy predicates

  • πsname, age(S)\pi_{\text{sname, age}}(S)πsname, age​(S) corresponds to SELECT sname,age

    • in real systems duplicates kept because of semantics and performance reason

  • σrating>8(S)\sigma_{\text{rating}>8}(S)σrating>8​(S) corresponds to WHERE rating>8

    • no duplicate generated

  • R∪SR \cup SR∪S corresponds to UNION

    • UNION ALL for keeping duplicates
    • schema compatibility required

  • R−SR- SR−S corresponds to EXCEPT

    • duplicates are not gernated
    • schema compatibility required

  • R×SR\times SR×S(no duplicates, compatibility not required)

  • ρ(OUT(1→sid1,4→sid2),R1×S1)\rho(OUT(1\rightarrow sid1, 4\rightarrow sid2), R1\times S1)ρ(OUT(1→sid1,4→sid2),R1×S1)(when R1 and S1 both have sid column)

  • R∩S=R−(R−S)R \cap S = R-(R-S)R∩S=R−(R−S)



  • Theta joins(⋈θ\bowtie_\theta⋈θ​): cross product followed by selection on θ\thetaθ

    • equi-join: theta join with theta being conjuction of equality(ANDed equals predicates)
      • natual join(⋈\bowtie⋈): equi-join on all matching column names

  • R⋈θS=σθ(R×S)R\bowtie_\theta S = \sigma_\theta (R\times S)R⋈θ​S=σθ​(R×S)

  • R⋈S=s=πuniqueσequal matching fields(R×S)R\bowtie S = s=\pi_\text{unique}\sigma_\text{equal matching fields}(R\times S)R⋈S=s=πunique​σequal matching fields​(R×S)

  • good join algorithms are needed to avoid cross product

  • example: R1⋈sid=sidS1R1 \bowtie_{sid=sid} S1R1⋈sid=sid​S1



  • group by/aggregation(γ\gammaγ)
    • γage,AVG(rating)(S)\gamma_\text{age,AVG(rating)}(S)γage,AVG(rating)​(S)
    • γage,AVG(rating),COUNT(*)>2(S)\gamma_\text{age,AVG(rating),COUNT(*)>2}(S)γage,AVG(rating),COUNT(*)>2​(S) (with Having clause)
